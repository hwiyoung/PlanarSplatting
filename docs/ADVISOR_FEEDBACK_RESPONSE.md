# 지도교수 피드백 대응 및 연구 방향 조정안 (v3)

> 작성일: 2026-03-11
> 목적: 지도교수 피드백 4가지 질문에 대한 대응, ground class 문제, 기여 방향 조정, 실험 우선순위를 정리한다.
> 상태: 기여의 정확한 경계는 Phase 3-B'과 Phase 4 결과에 따라 확정. 본 문서는 실험 진행의 기반이다.

## 1. 지도교수 질문에 대한 대응

### Q1. 왜 텍스처(L_photo)를 고려하지 않는가?

**현재**: L_mutual의 ablation을 위해 L_photo 미포함. 이는 실험적 편의이며 최종 목표(재구축 품질 최대화)와 정렬되지 않음.

**대응**: L_photo를 기본 설정에 포함한다. PGSR, 2DGS 등 기하학적으로 우수한 Gaussian Splatting 연구들이 모두 L_photo를 핵심 손실로 사용하며, 특히 MVS 깊이가 희소한 벽면 영역에서 텍스처 매칭이 기하학을 보완할 수 있다.

**기존 실험(Phase 3-B)의 위상**: L_photo 미포함 상태에서 수행된 Phase 3-B의 7개 ablation 결과는, L_photo 포함 상태에서 그대로 유효하다고 가정할 수 없다. 따라서 기존 결과는 "L_photo 미포함 조건에서의 예비 분석"으로 위치시키고, L_photo 포함 상태에서 baseline을 재확립한 후 L_mutual ablation을 재수행한다. 기존 Phase 3-B의 경로 1/2 진단은 재실험 설계의 논리적 근거로서 가치를 유지하며, "이 분석에 기반하여 L_photo 포함 조건에서 재실험을 설계했다"는 연결로 사용한다. L_photo 미포함 조건을 별도로 유지하여 비교할 필요는 없다.

### Q2. L_mutual이 CityGML 생성에 실제로 중요한가?

**현재**: L_sem만으로 mIoU=0.810 달성. L_mutual 추가 시 오히려 악화. CityGML 품질에서 프리미티브 병합, building instance 분리, 위상적 유효성이 L_mutual보다 더 직접적으로 중요할 가능성.

**대응**: L_mutual을 연구의 유일한 핵심 기여로 두지 않는다. L_mutual의 실제 효과는 실험으로 확인해야 하며, 결과에 따라 기여의 범위가 결정된다. 효과가 확인되면 핵심 기여의 하나가 되고, 효과가 제한적이면 "작동 조건 분석" 수준의 기여로 범위가 조정된다.

### Q3. Phase 4에서 mesh를 활용하면 어떨까?

**문제 인식**: Gaussian Splatting의 최대 약점은 위상적 불완전성(구멍, 틈, 비밀폐). CityGML val3dity 검증의 최대 장벽.

**대응**: TSDF mesh를 추출하여, mesh의 watertight 특성으로 PlanarSplatting 프리미티브 사이의 위상적 문제를 보정한다. mesh를 CityGML의 중간 표현으로 쓰는 것이 아니라, 프리미티브는 유지하면서 mesh 정보로 프리미티브 사이의 틈과 겹침을 보정하는 방향이다. 예를 들어, TSDF mesh에서 프리미티브 경계의 교차선을 추출하여 인접 프리미티브의 경계를 정밀하게 결정하거나, 프리미티브가 커버하지 못하는 영역을 mesh에서 식별하여 보충한다. 이것은 기존 연구에 없는 접근이며, 독립적 기여 가능성이 높다.

### Q4. Splatting이 기존 방법보다 구체적으로 무엇이 좋은가?

**핵심 차별점**: 기존 기하학적 방법(City3D, PolyFit)의 재구축 과정은 "평면 검출 → 후보 면 생성 → 면 선택 → 의미론 부여"가 순차적이며, 각 단계의 오류가 다음 단계에 전파된다. 제안 방법에서는 이 재구축 과정 전체가 미분 가능 렌더링 최적화 하나로 통합되어, 재구축 과정 내에서의 오차 전파가 방지된다. COLMAP SfM/MVS(전처리)와 Phase 4 CityGML 변환(후처리)은 이 재구축 과정의 바깥에 위치하므로, "재구축 과정 내에서의 오차 전파를 방지한다"는 주장은 성립한다.

추가적으로, 기하학과 의미론이 동일한 학습 가능 파라미터(프리미티브)에 인코딩되므로, 기존 방법의 "기하학 완성 후 사후 분류" 대신 "통합 학습"이 가능하다. 또한 장면당 수 분의 학습 시간과 수천 개 수준의 컴팩트한 프리미티브 표현은 도시 규모 확장성을 제공한다.

**검증 필요**: 이 차별점들은 City3D(코드 공개, CityGML 지원)와의 동일 데이터 비교 실험으로 실증해야 한다.

## 2. Ground Class와 CityGML GroundSurface의 불일치

현재 연구의 "ground" = 이미지에 보이는 실제 지면(도로, 보도). 항공 이미지에서 관측 가능.
CityGML의 bldg:GroundSurface = 건물의 하부 폐합면(건물 풋프린트). 항공 이미지에서 관측 불가.

**따라서**: ground 프리미티브는 CityGML 모델에 직접 포함되지 않는다. bldg:GroundSurface는 roof 경계를 지면 높이로 투영하여 별도 생성한다.

**ground class의 목적**: 건물 프리미티브(roof/wall)를 더 정확하게 식별하기 위한 배경(context) class이다. "이것은 지면이지 건물이 아니다"라는 분류를 학습함으로써, roof와 wall의 분류 정확도가 향상된다. Ground 프리미티브는 CityGML 변환 시 필터링되며, 건물 모델에는 roof와 wall 프리미티브만 포함된다.

**스케치 수정 필요**:
- 3.1절: "바닥" → "지면(terrain)", CityGML GroundSurface와의 구별, ground class의 목적(context class) 명시
- 3.4절: GroundSurface 생성 방법 수정 (ground 프리미티브 직접 매핑이 아니라 roof 경계의 지면 투영으로 별도 생성)

## 3. 기여 방향 (실험 결과에 따라 확정)

기여의 정확한 경계와 서술은 실험 결과 후 결정한다. 현재 시점에서의 큰 방향 세 가지:

### 방향 A: 항공 이미지 기반 CityGML LOD2 자동 생성
평면 프리미티브 표현과 의미론 통합을 통해, 항공 이미지에서 CityGML LOD2를 생성하는 것이 기존 방법(기하학적 파이프라인, CNN+후처리) 대비 어떤 장점을 가지는지를 실증한다.

이 방향의 학문적 기여는 "파이프라인을 구축했다"가 아니라, "평면 프리미티브 표현이 CityGML LOD2와 구조적으로 대응하며, 의미론 통합이 면 병합과 building instance 분리를 가능하게 한다"는 것이다. 즉, 표현의 선택과 학습 방식이 최종 목표에 왜 적합한지를 보여주는 것. City3D 등과의 비교가 핵심 근거.

### 방향 B: 의미론-기하학 동시 최적화를 통한 상호보완
기존 기하학 기반 방법에서는 전 단계 오차가 다음 단계에 전파되어, 의미론이 기하학을 교정할 수 없다. Neural 기반 동시 최적화에서는 의미론이 기하학에 피드백되는 구조적 가능성이 생긴다. 다만 이 피드백은 L_sem만으로는 발생하지 않으며(gradient 격리: L_sem → f_i only), L_mutual이 있어야 R_i에도 gradient가 흐른다. 따라서 이 방향은 실질적으로 L_mutual의 효과에 의존한다.

L_mutual의 실제 효과는 L_photo 포함 조건에서의 재실험으로 확인하며, 결과에 따라 기여의 범위가 결정된다. 효과가 확인되면 핵심 기여의 하나가 되고, 효과가 제한적이면 "동시 최적화의 구조적 가능성을 설계하고 작동 조건을 분석했다" 수준의 기여가 된다.

### 방향 C: Mesh의 watertight 특성을 활용한 토폴로지 개선
Gaussian Splatting의 최대 약점(위상적 불완전성)을 TSDF mesh의 watertight 특성으로 보정하여, val3dity를 통과하는 CityGML 모델을 생성한다. 프리미티브 사이의 틈과 겹침을 mesh 정보로 보정하는 접근. 기존 연구에 없다.

### 기여 확정 시점
- 방향 A: Phase 4 프로토타입 + City3D 비교 후
- 방향 B: L_photo 포함 조건에서의 L_mutual 재실험 후
- 방향 C: Phase 4에서 mesh 기반 토폴로지 보정 구현 후

## 4. 조정된 실험 우선순위

### 1순위: Phase 4 프로토타입 + L_photo 포함 실험 (병행)

Phase 4까지 끝까지 가보는 것이 핵심 목표이다. 두 작업을 병행한다.

**Phase 4 프로토타입** (새로운 코드 작성 필요, 2-3주 예상): 현재 (c) Independent 결과에서 CityGML 변환 시도. 프리미티브에서 직접 gml:Polygon 생성, building instance 분리, TSDF mesh로 위상 보정, val3dity 검증. 작동 여부 확인이 목적.

**L_photo 포함 실험** (config 변경 수준, 수일): (c') Independent+Photo를 학습하여 L_photo 포함 baseline 확립. 기하학 품질(Depth MAE, Normal cos)의 변화 확인.

### 2순위: L_mutual 재실험 + City3D 비교 (2-3주)

**L_mutual 재실험**: L_photo 포함 상태에서 (d') Joint+Photo 학습. 기존 Phase 3-B의 경로 1/2 진단에 기반한 d-masked 등 원인 분리 조건도 L_photo 포함 상태에서 재수행.

**City3D 비교**: 동일 성수동 데이터(COLMAP 포인트 클라우드)에서 City3D 실행. 기하학적 정확도, 의미론적 분류(City3D의 사후 휴리스틱 vs 제안 방법의 통합 학습), CityGML 유효성, 처리 시간 비교.

### 3순위: 확장 및 보완

**Mesh 기반 토폴로지 보정 고도화**: Phase 4 프로토타입 결과에서 위상적 문제를 진단하고, TSDF mesh 기반 보정 방법을 설계 및 구현.

**L_mutual 확장**: 재실험 결과에 따라 L_height, confidence-aware 등.

**추가 데이터셋**: UrbanScene3D 등에서 일반화 검증.
